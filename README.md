## Reflexión sobre el control de versiones
El control de versiones en GitHub permite mantener un historial claro de los cambios realizados, facilitando la colaboración y la trazabilidad de los documentos técnicos. Gracias a los commits descriptivos y ramas organizadas, es posible identificar el progreso del proyecto, corregir errores sin perder información y mejorar continuamente el sistema. Esta práctica asegura transparencia, eficiencia y calidad en el desarrollo de software.
